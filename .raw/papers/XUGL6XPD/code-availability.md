# Code Availability

## Verbatim statement

> Full methods and model formulation are available in supplemental information sections, model formulation and thermal energy storage (TES) constraints and data. Detailed code and data are available at https://doi.org/10.5281/zenodo.10619868. For detailed instructions on running model, please contact the corresponding author.

## Structural classification

| Question | Classification | Evidence |
|---|---|---|
| Has code DOI? | Yes | Evidence: The statement gives `https://doi.org/10.5281/zenodo.10619868` for detailed code and data. |
| Has repository link? | Yes | Evidence: Zenodo is named as the code and data archive. |
| Has on-request clause? | Yes | Evidence: Running instructions are available by contacting the corresponding author. |
| Source Data file? | Not as a Zotero child attachment | Evidence: Zotero has one PDF child and no separate code or data child. |
| FAIR-compliance assessment | Partial | Evidence: Archived code and data are citable, but run instructions are not fully self-contained in the availability statement. |

## Lesson for my own CA statements

Lesson: If a model is built in Pyomo or another optimization framework, publish the environment, solver version, run command, and example config alongside the archive. The availability statement should let a reader start without emailing the author.
